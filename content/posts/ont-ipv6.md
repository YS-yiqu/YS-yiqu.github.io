## 故事开头

桥接搞不定之后，退而求其次——光猫继续拨号，华硕做二级路由。

但问题来了：光猫拿到的 IPv4 地址是 `10.70.96.219`——这是运营商 CGN 内网地址。

```
你的光猫 IPv4：10.70.96.219
公网用户能访问到吗？→ 不能。
因为 10.x.x.x 是 CGN，100 个用户共享一个公网出口。
```

做端口转发？白做——外面访问不到你。

但这台光猫同时拿到了公网 IPv6 地址。这是关键。

---

## 原理：CGN 和 IPv6 的区别

### CGN（运营商级 NAT）

```
互联网用户 → 公网 IP (123.x.x.x) → CGN 网关 → 你的光猫 (10.x.x.x)
                                        │
                                  同一公网 IP 下有 100 个用户
                                  外面的请求不知道该转发给谁
                                  除非你向运营商申请端口映射（一般不给）
```

### 公网 IPv6

```
互联网用户 → 你的 NAS (2408:8207:5451:46e0:xxxx)
                │
          全球唯一地址，不需要 NAT
          数据包直接到达你的设备
          不需要端口转发，只需要防火墙放行
```

IPv6 的设计就**没有 NAT**。每个设备直接拿全球唯一的公网地址。这对终端用户来说是极大的解放。

---

## 方案：光猫拨号 + 华硕 DHCPv6-PD + 设备直连

### 架构

```
光纤 → 光猫（PPPoE 拨号，拿到 /60 IPv6 前缀）
         │
         ├── 华硕 WAN（DHCP 拿 IPv4 + DHCPv6-PD 拿 /62 前缀）
         │     └── 华硕下所有设备：公网 IPv6 + 内网 IPv4
         │
         └── NAS（直连光猫，拿到公网 IPv6）
```

### 关键概念：DHCPv6-PD（前缀委派）

运营商给光猫分配一个 `/60` 的 IPv6 前缀。华硕用 DHCPv6-PD 协议向光猫申请，分到一个 `/62` 子前缀。

```
运营商 → 光猫：这里有个 /60 位置给你用
光猫   → 华硕：分你一个 /62，你管你下面的设备
华硕   → 手机：你的是 2408:8207:5451:46e4:xxxx
```

每台设备都有独立的公网 IPv6 地址，全世界都能直达。

---

## 操作步骤

### 光猫侧

- 保持 PPPoE 路由模式（不变）
- 开启 IPv6（通常默认已开）

### 华硕侧

| 设置 | 值 |
|------|-----|
| WAN 类型 | 自动获取 IP |
| IPv6 类型 | **Native** |
| 获取方式 | **DHCPv6-PD** |

保存，等 30 秒。华硕应该拿到 `2408:xxxx` 开头的 IPv6 地址。

### NAS 侧

NAS 插在华硕 LAN 口（或光猫 LAN 口也行），自动获取 IPv6 地址。

### 验证

- `test-ipv6.com` → IPv6 得分 10/10 ✅
- 手机流量 ping NAS 的 IPv6 → 通 ✅
- 手机流量访问 NAS 网页后台 → `https://[IPv6地址]:9443` → 通了 ✅

---

## 端口怎么开

IPv6 不需要端口转发。每台设备有独立公网地址，只需在华硕防火墙放行：

```
华硕后台 → 防火墙 → IPv6 防火墙 → 新增规则
→ 允许 → 填写设备的 IPv6 地址 → 填写端口 → 保存
```

---

## 局限

| 能做 | 不能做 |
|------|--------|
| NAS 公网访问 ✅ | IPv4 端口转发（CGN 限制）❌ |
| 远程桌面 ✅ | 游戏主机 NAT 开放（要看游戏是否支持 IPv6）|
| 文件共享 ✅ | 某些老旧软件（只认 IPv4）|
| 网站/服务部署 ✅ | 微信小程序回调（要求 IPv4）|

大部分现代应用都已经支持 IPv6。如果遇到只认 IPv4 的服务，用 Tailscale 的 IPv4 over IPv6 隧道能解决。

---

## 一句话

> IPv6 时代已经到了你的光猫上。
> 别再纠结公网 IPv4 了——你手上就有一个每台设备都能用的公网。
> 只需要打开它。
