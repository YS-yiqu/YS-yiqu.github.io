## 事情是这样的

某天同事发来一个 `.docx` 文件，双击打开，Word 弹了个窗：

> *"Word 发现无法读取的内容。是否恢复此文档的内容？"*

点击"是"——然并卵 😅

更诡异的是，这个文件**足足 3MB**，但里面的内容明明就是几行字加一张表格……事出反常必有妖。

## 根因：.docx 到底是什么？

很多人以为 `.docx` 是一个"文件格式"，其实它是——**一个穿了马甲的 ZIP 压缩包** 🎭

你可以试试：把 `.docx` 改成 `.zip`，双击解压，里面是一堆 XML 文件。Word 打开文档时，其实是按一个叫 **OPC（Open Packaging Convention）** 的标准来读这个 ZIP 包的。

### 打个比方 🚚

> 把 .docx 想象成一个**快递包裹**：
>
> - `[Content_Types].xml` = **快递单**（说明箱子里装了什么）
> - `_rels/.rels` = **开箱指南**（告诉 Word 主文档在哪）
> - `word/document.xml` = **真正的货物**（你的内容）
>
> Word 收货时，第一步就是找**快递单**和**开箱指南**。如果找不到——直接退货 ❌

### 这文件出了什么问题？

这个文件的 ZIP 结构长这样：

```
快递包裹（ZIP根目录）
├── 📦 另一个快递.zip          ← ❌ 杂货
├── 📦 某模板.docx             ← ❌ 杂货
├── 📂 参考模板/               ← ❌ 杂货
├── 📂 素材/                   ← ❌ 杂货
└── 📂 output_template/        ← 咦？东西在这里面
    ├── ✅ [Content_Types].xml ← 快递单被塞进小抽屉了！
    ├── ✅ _rels/.rels         ← 开箱指南也被塞进去了！
    └── ✅ word/               ← 货物在这里
```

**Word 的逻辑**：快递单应该在包裹表面 → 找不到 → 拒收 🚫

**实际状况**：快递单被塞在了包裹内部的一个小抽屉里，Word 根本没翻到。

---

## 怎么修？思路比命令重要

### 核心思路

就一句话：**把正确的文档内容提到 ZIP 根目录来。**

提取那个藏着文档的子目录，把它下面的所有文件**提一级**放到 ZIP 根目录，重新打包。

### 技术要点

1. **必须保留的文件**（放到 ZIP 根目录）：
   - `[Content_Types].xml` ← Word 最先找它
   - `_rels/.rels` ← Word 接着找它
   - `word/document.xml` ← 你的正文
   - `word/_rels/document.xml.rels` ← 正文的关系引用
   - 其他 `word/styles.xml`、`word/settings.xml` 等配套文件

2. **多余的东西别带**：
   - 内嵌的其他 ZIP 文件 → 删
   - 内嵌的其他 .docx 模板 → 删
   - 素材目录、参考目录 → 删
   - 这些只会让文件体积虚胖，Word 也不认

### 原理：为什么这样能修好？

因为 OPC 标准的解析入口是死的——Word 一定先从 ZIP 根目录读 `[Content_Types].xml`，然后通过 `_rels/.rels` 找到主文档。只要这两个文件在根目录，文档内容不管有没有图片、表格、样式，Word 都能顺藤摸瓜全部找到。

## 经验教训

### 1. 不是所有 .docx 都能用 Word 生成

用脚本自动生成 .docx 时（比如 Python 的 `zipfile` 手动打包），最容易踩的坑就是——**路径层级不对**。

核心原则：**打包时入口文件必须在根目录，不能有文件夹前缀。**

### 2. 判断 .docx 健康状态的土办法

```python
import zipfile

with zipfile.ZipFile('可疑文件.docx', 'r') as zf:
    files = zf.namelist()

print('[Content_Types].xml 在根目录:', '[Content_Types].xml' in files)
print('_rels/.rels 在根目录:', '_rels/.rels' in files)
```

两个都是 `True` → 大概率没问题
任何一个 `False` → 恭喜你，踩坑了 😂

### 3. 文件大小是最好的警报器

纯文字表格的 .docx，正常就是 10~30KB。如果你的文档只有几行字却有几十 MB——检查一下 ZIP 里面是不是塞了奇怪的东西。
