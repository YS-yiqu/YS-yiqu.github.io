想拥有一个自己的网站？我就是这么开始的。这篇文章记录了我从"不知道博客是什么"到"本地访问网站"的完整过程。

---

## 背景

看到别人的个人博客很酷，我也想有一个。但当时我对这些东西一无所知——不知道什么是框架、什么是 Git、什么是命令行。

网上教程很多，但大部分默认读者"应该懂一些基础"。作为一个真·零基础选手，我踩了不少坑。这篇记录希望能帮到跟我一样从零开始的人。

---

## 核心概念：一个比喻

在动手之前，先用一个比喻说清楚几个关键概念：

| 概念 | 比喻 | 说明 |
|------|------|------|
| 框架（Hexo） | 房子模板 | 别人写好的代码，你拿过来改改配置就能用 |
| 运行环境（Node.js） | 施工工具 | 框架需要"工具"才能跑起来 |
| Git / GitHub | 云存储 | 把本地的代码备份到云端，还能发布成网页 |
| 主题 | 装修风格 | 给房子换外观，不改结构 |

理解了这些，后面操作就不会一头雾水了。

---

## 实际操作

### 第一步：安装 Node.js

Hexo 依赖 Node.js 运行。去 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本，一路点"下一步"就行。

装完后在 CMD 输入 `node -v`，能看到版本号就装好了。

### 第二步：安装 Hexo

打开命令行（右键桌面 → Git Bash Here），输入：

```bash
npm config set registry https://registry.npmmirror.com/
npm install -g hexo-cli
```

> ⚠️ 第一行是配置国内镜像，不然下载会很慢甚至失败。这个坑我踩了很久才搞明白。

### 第三步：初始化博客

在你想放博客的文件夹里打开命令行，输入：

```bash
hexo init
hexo server
```

打开浏览器访问 `http://localhost:4000`，你就能看到自己的网站了！

**就这么简单？** 是的，一个能本地访问的网站只需要这几步。后面的美化、上线都是在此基础上加东西。

---

## 踩坑记录

### 坑1：npm install 卡住不动

**现象**：命令行一直转圈，没有反应。

**原因**：npm 默认从国外服务器下载，国内网络慢。

**解决**：换淘宝镜像 `npm config set registry https://registry.npmmirror.com/`

### 坑2：Git Bash 里复制粘贴不好使

**原因**：Git Bash 的快捷键不一样。

**解决**：用 `Shift + Insert` 粘贴（不是 `Ctrl + V`）。

### 坑3：不知道"文件夹"选在哪

**理解**：你打开命令行的那个文件夹，就是"工作目录"。命令会在那个文件夹里创建文件。建议直接在你想放博客的地方右键 → Git Bash Here。

---

## 工具选择

我用的是 **WebStorm**（一个集成开发环境），它在内部就能运行命令行，不用单独装 Git。VS Code 也可以，只是需要自己装 Git 和配置插件。

如果跟我一样不想折腾配置，WebStorm 开箱即用更方便。如果是学生还可以免费申请教育版。

---

## 经验总结

1. **先理解概念再动手**：知道"框架是什么"比"怎么输命令"更重要
2. **善用镜像**：国内下载各种开发工具，换镜像能省大量时间
3. **不要求完美**：先跑通一个最简单的版本，再逐步优化
4. **记录踩坑**：每个报错都是经验，记下来下次就不会懵

从这篇博客开始，我的网站搭建之路就这样开始了 🚀
