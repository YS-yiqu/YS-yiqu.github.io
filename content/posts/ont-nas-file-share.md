## 故事开头

装好 Tailscale 之后发现一个神奇的事——用 Tailscale 的 `100.x.x.x` 地址，在 Windows 资源管理器里竟然能直接打开 NAS 上的文件夹。

```
\\100.64.161.84 → 回车 → NAS 上所有共享文件夹全出来了 🤯
```

然后右击 → 映射网络驱动器 → 选 Z: → 完成。

于是「此电脑」里多了一个 Z 盘，双击进去就是远在几十公里外的 NAS。

---

## 原理：SMB 协议 + Tailscale 隧道

SMB（Server Message Block）是 Windows 自带的文件共享协议，本来设计用在局域网里。

Tailscale 把远在天边的设备拉进一个「虚拟局域网」，SMB 就以为对方在同一个路由器下面，于是直接能访问。

```
物理上： 你家 NAS ←── 互联网 ──→ 公司笔记本

Tailscale 虚拟网络上：
   NAS (100.64.161.84) ←── WireGuard 加密隧道 ──→ 笔记本 (100.102.38.78)

SMB 协议看到的： 哦，100.64.161.84 在跟我同一个局域网
```

## 操作步骤

### 1. NAS 侧：确认 SMB 已开启

绿联 NAS 后台：

```
控制面板 → 文件服务 → SMB → 开启
→ 共享文件夹管理 → 把要共享的文件夹设为可见
```

### 2. Windows 侧：访问 NAS

打开任意文件夹（此电脑），在**地址栏**输入：

```
\\100.64.161.84
```

回车。输入 NAS 的账号密码后，所有共享文件夹出现。

### 3. 映射为 Z 盘（一劳永逸）

```
此电脑 → 右键 → 映射网络驱动器
→ 驱动器：Z:
→ 文件夹：\\100.64.161.84\你的共享文件夹
→ ✅ 登录时重新连接
→ 完成
```

以后打开此电脑，Z 盘就一直在那里。

## 常见操作

### 换账号登录 / 清除记住的密码

如果 Windows 记住了错误的登录凭据：

```bash
# 断开当前连接
net use \\100.64.161.84 /delete
```

下次访问时重新输入账号密码。

### 图形界面清除

```
控制面板 → 用户账户 → 凭据管理器 → Windows 凭据
→ 找到 100.64.161.84 → 删除
```

### 手机访问

| 系统 | 方法 |
|------|------|
| iOS | Files App → 右上角 ... → 连接服务器 → `smb://100.64.161.84` |
| Android | Solid Explorer → 新建 SMB 连接 → 填地址 |

## 速度

文件共享的带宽取决于你的宽带上传速度。通常：

```
同城：20-50 MB/s（宽带上传 100Mbps）
跨省：5-20 MB/s

4G 电影从 NAS 拉到笔记本：约 2 分钟 ✅
同样的电影用百度网盘：非会员 3 小时 ❌
```

## 对比其他方案

| 方案 | 速度 | 方便程度 | 费用 |
|------|------|---------|------|
| NAS 网页上传下载 | 慢（HTTP 单线程） | 要打开浏览器 | 免费 |
| 百度网盘 | 极慢（限速） | 网页 | 会员 ¥30/月 |
| 公网直连（端口转发+IPv6） | 快 | 要配置端口 | 免费 |
| **Tailscale + SMB** | **极快（P2P）** | **像本地硬盘** | **免费** |

---

## 注意事项

- SMB 不能共享整个盘，只能共享指定文件夹（微软的设计，吐槽归吐槽）
- 你在 NAS 后台**新建共享文件夹**来选你的文件夹
- 如果访问时提示「网络路径未找到」，先 `tailscale status` 确认 NAS 在线
