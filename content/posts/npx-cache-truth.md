一句话背景：帮朋友研究一个 AI 编程工具的安装方式时，意外发现 npx 的行为早在 2020 年就变了，而绝大多数人的认知还停在 2017 年。

---

## 现象：一个理所当然的"常识"

朋友说："npx 每次启动都要重新下载，太慢了，不如全局安装。"

这不是他一个人的看法。网上搜"npx vs 全局安装"，铺天盖地的观点都是：

> npx 把包下载到临时目录 → 执行完 → 删掉 → 下次又要重新下

听起来太合理了。毕竟阮一峰老师 2019 年的教程也是这么写的："npx 将 create-react-app 下载到一个临时目录，使用以后再删除。"

但——要是我告诉你，这个行为在 2020 年就已经变了呢？😅

---

## 根因：npm@7 悄悄重写了整个 npx

### 故事要从 2017 年说起

npx 诞生于 npm@5.2.0（2017 年 7 月），创造者是 npm 当时的维护者 Kat Marchán。它的初心很简单——解决三个痛点：

| 痛点 | 当时的解决方案 | npx 方案 |
|------|-------------|---------|
| 调用项目本地模块很麻烦 | `./node_modules/.bin/mocha --version` | `npx mocha --version` |
| 脚手架每次都要全局安装 | `npm i -g create-react-app` → 用完就忘 | `npx create-react-app my-app` |
| 多项目需要不同版本的同一个工具 | 全局只有一个版本，经常冲突 | `npx tool@3.x`、`npx tool@4.x` |

那个时代的 npx，确实是"下载 → 执行 → 删除"的流水线。这是它的卖点——**零残留、零污染**。

### 2020 年的转折

npm@7.0.0（2020 年 10 月）做了一个重大决定：**废弃独立的 `npx` 包，把 npx 重写为 `npm exec` 的内置命令。**

打个比方，npx 从一个"外卖小哥"变成了"自建厨房"——

- **外卖小哥模式**（npm@5-6）：你想吃什么？我叫个外卖，吃完连盒子都扔了。下次想吃了再叫。
- **自建厨房模式**（npm@7+）：外卖吃过一次？留下食谱和食材放冰箱。下次想吃了直接从冰箱拿，不用再点外卖。

这个"冰箱"就是 `~/.npm/_npx/` 缓存目录。

**只有在版本变化、或缓存被清理时，它才会重新从 registry 拉取。**

---

## 为什么你还在「被骗」？

因为好内容寿命太长，反而成了"僵尸信息"。

2019 年阮一峰那篇《npx 使用教程》影响了一整代开发者。写得确实好——清晰、实用、覆盖全面。但它发布于 npm@6 时代。

之后 npm@7 带来了缓存机制，但没有一篇文章能做到同样的传播力来"纠正"这个认知。

于是，绝大多数人脑子里还是那个"每次重新下载"的 npx。

不止 npx 有这种现象，curl 的 `--parallel`、Python 的 `walrus operator`、Git 的 `switch/restore`... 工具在进化，教程却容易定格在它们被写下的那一刻。

这也是为什么要定期审视自己的"常识"——你三年前学的东西，可能早就不是那样了 😄

---

## 这个发现改变了什么？

以前讨论"npx 还是全局安装"时，天平是倾斜的：

```
全局安装：方便，但容易忘记更新
npx：    每次都最新，但是慢
```

现在天平被拉平了：

| 维度 | 全局安装 | npx（2025 实际） |
|------|---------|----------------|
| 首次启动 | 需安装 | 需下载 |
| 后续启动 | 秒开 | **缓存秒开** ✅ |
| 自动更新 | ❌ 手动 `npm update -g` | ✅ 自动最新 |
| 版本灵活 | 全局一个版本 | 随便指定版本 |
| 全局污染 | 有 | 无 |

加上缓存后，**npx 在启动速度上已经不输全局安装**，同时还自带"永不陈旧"的被动技能。

**当然全局安装没死**——`pnpm`、`pm2`、内网服务器、生产运行时依赖这些场景，全局/项目安装仍然是正确选择。但如果你纠结"一个 npm 工具要 npx 还是全局装"，2025 年的答案是：**优先 npx，除非你有明确的理由不这么做。**

## 自己验证一下？

不一定相信我说的，自己动手试试看（注意环境差异 👇）：

**Linux / macOS：**

```bash
# 跑两遍看看速度差
npx cowsay hello    # 第一次
npx cowsay hello    # 第二次 — 明显快

# 看看冰箱里有什么
ls ~/.npm/_npx/
```

**Windows：**

```powershell
# 跑两遍看看速度差
npx cowsay hello    # 第一次
npx cowsay hello    # 第二次 — 明显快

# 看看冰箱里有什么
dir %LocalAppData%\npm-cache\_npx
```
