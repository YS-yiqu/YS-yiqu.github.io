黑进自家光猫：宽带密码提取记

买了新路由器想改桥接，宽带密码忘了？不用打客服，从光猫后台下载配置文件，3 秒提取账号密码。

故事开头#

买了台新路由器，想改桥接。光猫后台一点——宽带账号显示 9900******，密码是六个黑点。

“密码是啥来着？"——装机师傅给的纸早就找不到了。

打 10010？客服能告诉你账号，但密码得「重置」，重置完光猫自己也要重新配。

于是决定：自己动手拿回来。

原理：光猫的配置到底存哪了#

光猫本质是一台跑 Linux 的小电脑，所有配置（宽带账号密码、WiFi 密码、远程管理参数、电话设置……）都存在一个 TR-069 协议的 XML 数据库里。

TR-069 是运营商远程管理光猫的协议，定义了光猫内部所有可配置参数的数据模型。你的宽带账号密码在模型里叫 WANPPPConnection.UserName 和 WANPPPConnection.Password。

而这个数据库，在光猫后台的「配置上传」功能里，可以下载。

过程#

第一步：登录光猫#

光猫的登录页面用 AES 加密用户名密码。但 AES 的密钥写在网页的 JavaScript 里，任何人都能看到：

1
2

var key = "1111111111111111";   // AES 密钥（是的，16个1）
var iv  = "0000000000000000";   // IV（16个0）

安全建议：用 1111111111111111 当密钥 ≈ 你家大门密码是 password123。拿到后记得把默认密码改了。

默认管理员账号 CUAdmin，密码 CUAdmin。用这个 AES 加密后 POST 登录。

第二步：下载配置文件#

登录后访问「管理 → 系统管理 → 配置上传」页面。页面上有一个「下载用户配置」按钮，点击后 JavaScript 会做三件事：

① 提取页面里的 UPLOAD_SESSION_TOKEN
② POST 到 checkSessionToken.gch 验证
③ POST 到 getpage.gch?pid=101 下载 519KB 的完整配置 XML

这个接口没有权限校验——只要你能登录后台，就能下载。

第三步：提取凭据#

下载的 XML 里搜索 UserName 和 Password，就能找到宽带账号密码：

1
2

<DM name="UserName" val="宽带账号"/>
<DM name="Password" val="宽带密码"/>

用 Node.js 写的脚本，整个流程自动化，3 秒完成。

为什么这么容易？#

光猫 Web 后台的安全设计有四个问题：

问题影响
AES 密钥硬编码在前端 JS任何人看网页源码就能获取
默认密码没有强制修改多数人用的是出厂默认
配置下载接口没有二次验证登录后台就能下载全部配置
密码明文存储在 XML 中下载后直接可读

本质上，厂商把家用光猫当成「内部设备」而非「安全边界」。但任何连上你 WiFi 的人，用同样的方法都能拿到你的宽带密码——这才是真正的风险。

拿到之后还能怎么保护#

  1. 改光猫默认密码。这个最重要，改了之后别人就没法用上述方法拿你的配置。
  2. 关掉 WAN 侧 Web 管理。防止从公网访问光猫后台。
  3. 定期检查配置有什么变化。运营商可通过 TR-069 远程改你设置。

一句话#

光猫安全的薄弱环节不在加密算法，在你没改默认密码。 拿到密码只是开始——别让任何人用同样的方法拿到你的。
