桥接踩坑记：6 种方案全军覆没

拿到宽带密码后开始改桥接，结果 6 个小时 6 种方案全部失败。根因不是配置问题，是光猫固件的 VLAN 透传有 bug。

故事开头#

拿到宽带密码后，开始改桥接。

目标：光猫只做光电转换，华硕路由器拨号，拿公网 IP。

听起来简单——光猫后台改个模式就行。结果……6 个小时，6 种方案，全部失败。

原理：PPPoE 拨号的四步握手#

在你改桥接之前，先理解 PPPoE 怎么拨号的：

Step 1: 路由器发广播 → PADI（「喂！谁给我IP？」）
Step 2: 运营商回      → PADO（「我！这有IP给你！」）
Step 3: 路由器确认    → PADR（「就要你的！」）
Step 4: 运营商确认    → PADS（「给你IP了，用吧」）

路由模式下，步骤 1-4 在光猫内部走完，外面的设备不参与。

桥接模式下，步骤 1-4 的数据帧需要从光猫 LAN 口原封不动穿过光猫 CPU 到达光纤。

问题就出在「原封不动穿过」这一步。

六种尝试#

方案一：新建桥接 WAN 连接#

新建 WAN → 模式：桥模式 → 桥类型：PPPoE_Bridged → VLAN：不启用(untag) → 绑定 LAN4

华硕日志：

Timeout waiting for PADO packets

华硕发了 PADI，但 OLT 没回 PADO。光猫没有把发现帧透传出去。

方案二：VLAN 改为「改写」(tag)#

VLAN 模式：改写 → VLAN ID：3961

光猫把 LAN4 收的不带标签帧打上 VLAN 3961 → 发到光纤。同上，失败。

方案三：VLAN 改为「透传」#

VLAN 模式：透传 → 华硕 IPTV 页面设 VID 3961

由华硕负责打 VLAN 标签，光猫只做二层转发。同上，失败。

方案四：改原始 INTERNET 连接为桥接#

不新建，直接把原来那条 PPPoE 路由连接改成桥接。

同上，失败。

方案五：禁用旧连接 + 重启#

怀疑旧 pppd 进程占着线。禁用旧连接 → 重启光猫 → 新桥接启用 → 华硕拨号。

同上，失败。而且重启后旧连接又自动启用了（Tr-069 远程配置覆盖）。

方案六：配置文件修改 → 上传#

找到配置 XML 中的连接参数，手动改成桥接模式，上传到光猫。

ConnType="1" (PPPoE路由) → 改成 → ConnType="4" (PPPoE桥接)

上传成功，光猫接受配置。但——华硕依旧 Timeout waiting for PADO packets。

根因分析#

六种方案的共同失败点：PPPoE 发现帧没有从光猫 LAN 口到达光纤。

这意味着问题不在配置，而在光猫固件的桥接实现有 bug。可能的原因：

可能原因说明
硬件层面没有建立 VLAN 透传通道交换芯片没有把 VLAN 3961 绑定到 LAN4
pppd 的 zxic_plugin 有 bugZTE 私有插件在桥接模式下行为异常
运营商通过 TR-069 远程锁定禁止桥接模式

对于终端用户，这些都无法验证或修复。

教训#

  1. 不是所有光猫都支持桥接。烽火 SG659HZ 的当前固件版本就是有 bug。换其他型号的第三方光猫可能能解决，但需要 LOID 和 VLAN 信息。

  2. IPv6 可以替代桥接。如果你只需要公网访问服务（NAS、远程桌面），IPv6 公网地址 + Tailscale 组网的效果不亚于桥接。详见下一篇文章。

  3. 做任何改动前先备份配置。搞砸了用备份一键恢复，比打电话找运营商快一百倍。

如果必须桥接#

唯一靠谱方案：换光猫。

需要准备的信息（从当前光猫获取）：

  • LOID（逻辑标识，光猫后台 → 网络 → 宽带识别码设置）
  • VLAN 3961（INTERNET 连接）
  • PPPoE 账号密码（本文脱敏处理，你自己的光猫里有）

推荐第三方光猫型号：华为 HG8010H、MA5671a、诺基亚 G-010S-A（纯桥接 GPON ONU，200-500 元）。

一句话#

桥接不是光猫的标配功能——有些固件实现有 bug。 别在里面耗 6 小时，试两次不行就换方案。
